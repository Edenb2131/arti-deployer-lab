# Single Artifactory instance

The default topology: one Artifactory Pro container + one PostgreSQL container,
both on a shared bridge network `arti-deployer_net`.

## What gets created

| Component | Container | Host port | Purpose |
|---|---|---|---|
| Artifactory 1 | `artifactory1` | `${AF1_HTTP_PORT}` (legacy 8081), `${AF1_ROUTER_PORT}` (router 8082) | Main AF |
| Postgres for AF1 | `postgres-art1` | (internal only) | DB backend |

## Volumes

- `arti-deployer_postgres-art1-data` — Postgres datadir
- `arti-deployer_artifactory1-data` — `/var/opt/jfrog/artifactory`

Both survive `down` and are wiped by `reset`.

## Files bind-mounted in

- `config/art1/system.yaml` → `/var/opt/jfrog/artifactory/etc/system.yaml`
- `config/art1/artifactory.lic` (generated from `.env`) → `/var/opt/jfrog/artifactory/etc/artifactory/artifactory.lic`

## First login

```
URL:      http://localhost:8081/ui/
User:     admin
Password: password
```

You'll be prompted to change the password. After that, the admin token
generated by `configure-cot.sh` / `configure-ldap.sh` / `configure-keycloak.sh`
will use the new password — set it via `.env` if you want non-interactive flow
(see the configurator scripts).

## Tuning

`EXTRA_JAVA_OPTIONS` is `-Xms1g -Xmx4g` by default. Override in `compose/art1.yml`
if your tickets need more heap.

## Troubleshooting

**AF won't start, complains about license** — check `config/art1/artifactory.lic`
exists and is non-empty. The CLI generates it from `.env`; if `ARTIFACTORY_LICENSE`
is empty, you'll see a startup loop. Set it and run `./arti-deployer reset && up`.

**Permission denied on volume** — AF runs as UID `1030`. If you've ever run
this on Linux without the right permissions, wipe volumes with
`./arti-deployer reset`.

**`postgres-art1` healthy but AF stuck** — check
`./arti-deployer logs artifactory1`. Most common cause is wrong DB credentials
between `.env` and `system.yaml`. They must match — `PG_AF1_PASSWORD` in `.env`
equals `shared.database.password` in `system.yaml`. (TODO: render `system.yaml`
from a template at runtime so it picks up `.env` automatically.)
